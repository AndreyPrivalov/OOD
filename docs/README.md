# Documentation Map

`docs/` is the canonical documentation root for OOD.

Use the documentation layers like this:

- `spec.md` defines the master product scope and architectural direction.
- `AGENTS.md` defines implementation and review rules.
- `docs/product.md` defines shared product vocabulary and cross-feature business rules.
- `docs/design-system.md` defines shared UI behavior and visual rules.
- `docs/tech.md` defines technical invariants and architecture constraints.
- `docs/features/*.md` define feature-specific business logic.

Do not create or maintain duplicate sources of truth in `Docs/`, root markdown drafts, or local `rules.md` files.
