# Product

## Purpose

Этот документ фиксирует общую продуктовую логику OOD и служит каноническим cross-feature источником истины.

## Scope

- одно или несколько общих рабочих пространств;
- одно дерево работ на рабочее пространство;
- создание корневых и дочерних work items;
- inline-редактирование основных полей;
- оценки `overcomplication`, `importance`;
- workspace-scoped метрики влияния с отдельной конфигурацией на уровень рабочего пространства;
- агрегированные read-only значения у parent-узлов;
- перемещение внутри дерева и каскадное удаление ветки.

## Core Rules

- `title` не может сохраняться пустым;
- значения оценок ограничены диапазоном `0..5` или `null`;
- workspace может иметь собственный набор метрик влияния, включая пустой;
- значение метрики принимает одно из трёх состояний: `none`, `indirect`, `direct`;
- parent-узлы показывают только агрегированные metric values по потомкам;
- изменение бизнес-полей не должно неявно менять структуру дерева;
- текстовые поля `currentProblems` и `solutionVariants` считаются списками пунктов;
- work item принадлежит ровно одному workspace;
- перенос между workspace не входит в cleaned core.
