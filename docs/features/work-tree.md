# Work Tree

## Purpose

Этот документ описывает feature-specific правила дерева работ.

## Rules

- у workspace есть одно дерево work items;
- node может быть root или child;
- child имеет ровно одного valid parent;
- sibling order должен оставаться стабильным;
- перемещение узла перемещает всё его поддерево;
- invalid moves не должны применяться частично;
- удаление parent-узла каскадно удаляет всю ветку.
- для неструктурных правок дерева клиентские селекторы должны сохранять identity неизменённых строк и производных map-структур;
- после успешных optimistic `move/delete` не требуется обязательный полный refresh дерева, если локальное состояние уже консистентно.
- сворачивание/разворачивание ветки является только клиентским состоянием отображения и не изменяет tree-структуру или persistence-данные;
- активное сворачивание скрывает всех потомков выбранной строки до разворачивания этой строки пользователем.
- при пустом дереве таблица показывает центральную action-кнопку `Добавить работу`, которая создаёт первый root-элемент.
