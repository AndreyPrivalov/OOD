# Ratings

## Purpose

Этот документ задаёт бизнес-правила для core ratings.

## Rules

- используются три оценки: `overcomplication`, `importance`, `blocksMoney`;
- допустимые значения: целые числа `0..5` или `null`;
- оценки принадлежат конкретному work item;
- leaf-узлы показывают собственные оценки;
- parent-узлы показывают агрегированные read values;
- редактирование оценок не должно менять структуру дерева.
